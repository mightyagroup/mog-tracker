DATA METRICS COMPUTED
================================================================================

1. Pipeline Summary
   - Total leads across all entities
   - Active bids count
   - Awards count
   - Total pipeline value
   - Per-entity breakdown (leads, bids, awards, value)

2. Status Distribution
   - Lead counts per status per entity
   - All 9 statuses tracked: new, reviewing, bid_no_bid, active_bid,
     submitted, awarded, lost, no_bid, cancelled

3. Source Distribution
   - Lead counts by source type
   - All 8 sources: sam_gov, govwin, eva, emma, local_gov, usaspending,
     manual, commercial

4. Monthly Trends
   - Last 12 months of lead creation data
   - Separate line per entity
   - Month labels formatted as "MMM YYYY"

5. Fit Score Distribution
   - Leads in 4 buckets: 0-25, 26-50, 51-75, 76-100
   - Shows opportunity quality distribution

6. Win Rates
   - Per-entity calculation: awarded / (awarded + lost)
   - Percentage and raw counts

7. Average Time to Award
   - Days from created_at to awarded_date
   - Per-entity average

8. Deadline Proximity
   - Counts in 5 time buckets relative to current date
   - Flags urgent vs. planning opportunities

9. Top Agencies
   - Top 10 by lead count
   - Includes total value and average fit score
   - Useful for strategic focus

10. Category Breakdown
    - Leads per service category per entity
    - Value aggregation per category
    - Helps identify strongest service areas

11. Commercial Pipeline
    - VitalX-specific funnel view
    - Counts at each stage
    - Total pipeline value across all stages

COLORS & THEME

- Secondary backgrounds: #1F2937, #374151
- Text: white, #D1D5DB, #9CA3AF

  * Exousia: #D4AF37 (Gold)
  * VitalX: #06A59A (Teal)
  * IronHouse: #B45309 (Amber)
  * MOG: #D4AF37 (Gold)

RESPONSIVE DESIGN
- Mobile-first approach
- Summary cards: 1 column (mobile) → 4 columns (desktop)
- Charts: Full width, responsive height
- Tables: Card view on mobile
- Sidebar: Hamburger menu on mobile

PERFORMANCE
- Parallel API queries via Promise.all()
- Efficient Supabase filtering
- Client-side calculated metrics
- No persistent caching (fresh on each request)
- RLS policies enforced via Supabase auth
